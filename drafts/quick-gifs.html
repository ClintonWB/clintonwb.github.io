Quick GIFs from videos using FFmpeg

Clinton bradford | Mon 05 October 2020

Introduction

Background

These days, it is said that modern internet speech is similar to the Tamarians in the famous episode in season 5 of Star Trek: The Next Generation, Damok.

[]

This was recently called back to in an episode of Star Trek: Lower Decks, a show I have grown quite fond of, with the character Kayshon, a Starfleet officer and Tamarian.

[]

I would love to make quick reference to impactful moments of Lower Decks, and other shows I have come to love, in the manner of my people - short, silent, looped videos.

This document hopes to record my journey to be able to get acceptably good short animated GIFs in a quick way, from timestamps of videos. Ideally, this can serve as a quick reference for myself and others for common FFmpeg gif tasks.

Tools

FFmpeg is a powerful video and audio editing tool, with a variety of filters stackable and combinable to do everything I need to do. While I have in the past used Imagemagick and Pillow for a lot of my animating needs, I have been relying on their usage of FFmpeg under the hood, and it is time I learned a bit more about FFmpeg itself.

Plan

To make a GIF from a captured video, there are several steps:

  • Cut video to length
  • Extract frames
  • Resize/crop frames to desired image
  • Construct restricted GIF color palette
  • Flatten colors to palette
  • Composing images into final GIF

With optional:

  • Reversing images for smooth boomerang looping
  • Cutting out or duplicating frames for better effect
  • Add simple captions

I will document here how to do all of these in FFmpeg.

Methodology

To do these operations, we will primarily be using the -filter_complex option in FFmpeg. This option uses a series of filters, operations given in a sort of infix notation with inputs on the left, and parameters on the right, followed by outputs. Described in the filter documentation, these outputs and input "pads" can be left blank to automatically link between two adjacent filters. This can help for simple chains or brevity, but here I will be as explicit as possible.

Cutting with -ss and -t

The first step will be to cut the video to the correct length. Documented on the Seeking page, there are a number of considerations on relative placement listed there. Because I want to process them as fast as possible, With a start and clip length, we will use the -ss starttime seek command before the -i input command, and add the -t stoptime options to seek to a precise time. So far, the command we've built is:

ffmpeg -ss 00:17:30 -i episode.mkv -t 00:00:04 clip.gif

Which takes the 4-second long piece at 1 and clips it to a specific portion clip.gif:

[]

Pretty good results with just the basics, but we can do better.

Trimming Frames

Sometimes, nudging timestamps - even by small fractions of a second - isn't precise enough. A great start or end could be gained by extracting the frame images, and selecting, but for something quicker 

ffmpeg -ss 00:17:24.5 -t 5 -i episode.mkv -filter_complex "[0]trim=start_frame=1:end_frame=29,reverse,[0]concat[c];[c]crop=515:819:710:230,fps=10,scale=270:-1:flags=lanczos,split[s0][s1];[s0]palettegen[p];[s1][p]paletteuse" -loop 0 output.gif

[]

Extending on Frames

You can pad videos with t-pad! Write some more about this

ffmpeg -i in.mp4 -vf tpad=stop_mode=clone:stop_duration=2 out.mp4

Scaling

Our example is 25 megabytes. These videos do not need the full resolution of a TV show to convey meaning or look good, and for size concerns, we can resize them with the scale filter. For now, I'm just going to set the width of the video to 540 pixels, a common limit on platforms, and the height to -1, which auto-computes based on the existing ratio:

ffmpeg -ss 00:17:30 -i episode.mkv -t 00:00:04 -vf "scale=512:-1" clip.gif

[]

Framerate and Looping

ffmpeg -ss 00:17:30 -i episode.mkv -t 00:00:04 -vf "fps:10,scale=512:-1" -loop 0 clip.gif

[]

Color Palettes

When doing one-input-one-output operations, -vf works to use filters in sequence. However, sometimes we want to do operations on multiple video streams. One thing we can do with a complex filter is split the video out to a pallette generator, which uses the whole clip in order to generate a decent set of colors for the final GIF encoding:

ffmpeg -ss 00:17:30 -i episode.mkv -t 00:00:04 -filter_complex "fps:10,scale=512:-1,split[s0][s1];[s0]palettegen[p];[s1][p]paletteuse" -loop 0 clip.gif

[]

This can be a substantial improvement in GIF quality.

Conclusion

FFmpeg is a powerful tool - underpinning many of the free and commercial video viewing and editing tools we use - with an expressive API of its own. While I may hope to get these features in an easy to use VLC dialog, for most uses, personally I prize the power, customizability, and automatability that comes from knowing more about an interface like this.
